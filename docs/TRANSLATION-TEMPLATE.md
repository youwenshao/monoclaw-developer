# Translation Template for MonoClaw Wiki

> **用途：** 翻译 `docs/` 与 `hatch/docs/` 中的 Markdown 文档时，请以此模板为格式基准。
> **语言版本：** 简体中文 (zh-hans) / 繁体中文—香港 (zh-hant)

---

## 1. 文件命名

- **英文原文：** `example.md`
- **简体中文：** `example.zh-hans.md`
- **繁体中文：** `example.zh-hant.md`
- **位置：** 与英文原文同目录，不得另建子文件夹。

---

## 2. 文件头格式

每个译本必须在主标题后紧跟一个标准的译本信息块，格式如下：

```markdown
# 这里是译后主标题

> **译本信息**
> **原文：** `hatch/docs/technician-handbook.md`
> **基于提交：** `e909e16`
> **语言：** 简体中文（中国大陆）
> **最后更新：** 2026-05-14
```

**字段说明：**

| 字段 | 说明 |
|------|------|
| `原文` | 相对仓库根目录的英文源文件路径 |
| `基于提交` | 翻译所依据的英文版本 git short hash（`git rev-parse --short HEAD`） |
| `语言` | 完整语言名称，标明地区变体 |
| `最后更新` | 本次翻译或更新的日期，ISO-8601（YYYY-MM-DD） |

---

## 3. Markdown 元素处理规则

### 3.1 标题

- 保持与原文相同的层级结构（`#`、`##`、`###`）。
- 标题文字按 `TRANSLATION-GLOSSARY.md` 翻译。
- 若标题内含代码或路径（如 `~/.monoclaw`），保留原样。

### 3.2 段落与正文

- 完整翻译，不得遗漏句子。
- 长句可按中文阅读习惯拆分，但不得改变原意。
- 英文斜体或加粗在中文中对应**加粗**；斜体在中文排版中效果较弱，不建议单独使用。

### 3.3 代码块

```markdown
**保留原样**，块内注释可译，但命令、路径、变量名保持英文。

错误示例：
```bash
# 這是錯誤示範：不要把命令翻譯成中文
./安裝.sh
```

正确示例：
```bash
# 在目标 Mac 上运行安装脚本
./install.sh
```
```

### 3.4 行内代码

- 行内代码（`` `code` ``）**保留原样**。
- 若行内代码是纯英文说明性文字而非技术标识符，可在代码外给出中文解释。

### 3.5 表格

- 表头按术语表翻译。
- 单元格内容完整翻译。
- **命令、路径、环境变量**在表格中也保留英文。
- 翻译后请在 Markdown 预览器中检查表格边框是否对齐、是否中断。

示例：

```markdown
| 症状 | 原因 | 修复方法 |
|---|---|---|
| 安装后提示 `monoclaw: command not found` | 当前 shell 的 `PATH` 未包含 `~/.local/bin` | 新开一个终端窗口，或运行 `export PATH="$HOME/.local/bin:$PATH"` |
```

### 3.6 引用块（`>`）

- 警告、提示、重要说明等引用块完整翻译。
- 保留原文中的 emoji 或图标（如 `⚠️`、`✅`），若原文无则可不添加。

### 3.7 列表

- 有序列表与无序列表的引导文字完整翻译。
- 列表项内若含代码，按行内代码规则处理。

### 3.8 链接与锚点

- **外部 URL** 保留原样。
- **内部锚点**（如 `[See Section 5](#5-recovery--reruns)`）需确保锚点 ID 在翻译后仍然有效。
- GitHub Flavored Markdown 的标题锚点由标题文字自动生成。若中文标题的锚点导致链接失效，可用 HTML 显式指定锚点：

```markdown
## 恢复与重新运行 {#recovery-and-reruns}
```

**建议：** 优先保留英文锚点 ID，以维持与英文原文的链接兼容性：

```markdown
## 5. 恢复与重新运行 {#recovery--reruns}
```

## 4. 译注规则

若遇到以下情况，可在当前段落底部或页脚添加译注：

- 英文原文使用了难以直译的俚语或双关。
- 某术语在中文语境下含义模糊，需要额外解释。
- 英文原文存在已知问题或已过时，需要提醒读者。

译注格式：

```markdown
> **译注：** 此处英文原文 "bench" 指技术员的反复测试环境，并非性能基准测试（benchmark）。
```

## 5. 自检清单

提交翻译前，请逐项确认：

- [ ] 文件命名符合 `*.zh-hans.md` / `*.zh-hant.md` 规范
- [ ] 译本信息块包含原文路径、基于提交、语言、最后更新日期
- [ ] 所有术语与 `docs/TRANSLATION-GLOSSARY.md` 一致
- [ ] 产品名（MonoClaw、Hatch、Mona）保留英文
- [ ] 命令、路径、环境变量、代码块保留原样
- [ ] 品牌名保留英文
- [ ] 表格在 Markdown 预览器中渲染正常
- [ ] 内部锚点链接有效
- [ ] 简体中文使用规范大陆用语，繁体中文使用香港惯例
- [ ] 全文无遗漏段落

---

## 6. 快速参考：首次翻译流程

1. 阅读 `docs/TRANSLATION-GLOSSARY.md`，熟悉核心术语。
2. 复制英文原文为 `*.zh-hans.md` 和 `*.zh-hant.md`。
3. 替换主标题，插入译本信息块。
4. 逐段翻译正文，保留所有代码和路径。
5. 翻译完成后，对照本模板第 5 节逐项自检。
6. 在本地 Markdown 预览器（VS Code、GitHub、Cursor）中打开，确认渲染无异常。
7. 提交。
